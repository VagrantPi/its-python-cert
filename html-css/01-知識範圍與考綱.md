> **證照全名**：Certiport IT Specialist (ITS) — HTML and CSS
> **考試代碼**：ITS-301 / INF-301
> **發行單位**：Certiport（Pearson VUE 旗下）
> **台灣總代理**：碁峰資訊（GOTOP）
> **文件用途**：完整梳理考試涵蓋的知識範圍、六大領域權重與細部技能點，作為備考的主幹地圖。
> **調研日期**：2026-07（資料以官方 Objective Domains 與多方來源交叉驗證）

## 一、證照定位

ITS（IT Specialist，前身為 MTA 系列的接班認證）是 Certiport 針對**入門級／基礎程度**設計的國際認證。HTML and CSS 這一科用來證明應考者能夠：

> 「分析並撰寫語法正確的 HTML 與 CSS，運用 HTML 建構資料結構，並運用 CSS 建立與套用樣式。」
> （*analyze and produce syntactically correct HTML and CSS, structure data using HTML, and create/apply styles using CSS*）

- **難度定位**：入門～基礎。重點在**讀懂標記、理解語意結構、判斷樣式套用結果與除錯**，而非在考場即席刻出複雜網站。
- **建議先備經驗**：官方建議候選人具備**至少 150 小時**的 HTML/CSS 教學或實作經驗。
- **適合對象**：高中／大專學生、前端／網頁設計初學者、想要一張國際入門證照豐富學習歷程或履歷者、非資訊本科但需具備網頁製作基礎能力者。
- **與其他證照的關係**：
  - 前身為 Microsoft **98-383（Introduction to Programming Using HTML and CSS）**，兩者範圍高度重疊，舊教材可參考。
  - 屬 ITS 系列一員，與 ITS Python、ITS JavaScript 等並列，皆為入門級「基礎能力 + 國際認證背書」定位。

## 三、六大考試領域與權重（Objective Domains）

以下領域來自官方 Objective Domains 文件（ITS-301 / INF-301）。

> ⚠️ **權重說明**：Certiport **官方並未逐一公布各領域的權重百分比**。下表權重為**依各領域子目標數量推估的近似值**，僅供分配讀書時間參考，**一切以官方 Objective Domains 為準**。

| # | 領域（Domain） | 權重（近似） | 相對比重 |
|---|----------------|:----:|:--------:|
| 1 | HTML Fundamentals（HTML 基礎） | **10–15%** | 中低 |
| 2 | CSS Fundamentals（CSS 基礎） | **10–15%** | 中低 |
| 3 | Document Structure using HTML（以 HTML 建構文件結構） | **20–25%** | 高（占比最高之一） |
| 4 | Multimedia Presentation using HTML（以 HTML 呈現多媒體） | **10–15%** | 中低 |
| 5 | Webpage Styling using CSS（以 CSS 設定網頁樣式） | **20–25%** | 高（占比最高之一） |
| 6 | Accessibility, Readability, and Testing（無障礙、可讀性與測試） | **15–20%** | 中高 |

> 📌 **備考策略提示**：領域 3（文件結構）與領域 5（CSS 樣式）合計可達 40–50%，是必守的核心主幹，也是最能拉分的區塊；領域 6（無障礙與測試）占比不低且觀念導向，屬「投報率高」的區域；領域 1、2、4 為基礎與多媒體，內容單純但為理解後續領域的地基，不可略過。

## 四、各領域細部技能點（15 項官方子目標）

> 以下 15 項子目標對應官方 Objective Domains / CodeHS 標準框架的 standards，是最權威的「考什麼」清單。

### 領域 1：HTML 基礎（1.1–1.2）
- **1.1** 撰寫使用**中繼資料（metadata）元素**的標記
  → `script`、`noscript`、`style` 元素；`link` 元素；`meta` 標籤（`charset` 編碼、`keywords`、`viewport`、`description`）。
- **1.2** 撰寫**格式良好（well-formed）**的頁面標記
  → `DOCTYPE` 宣告、`html` / `head` / `body` 結構、正確語法與結尾標籤、常用字元符號（HTML entities，如 `&amp;`、`&lt;`、`&nbsp;`）。

### 領域 2：CSS 基礎（2.1–2.2）
- **2.1** 分析並實作**行內（inline）、內嵌（internal/embedded）與外部（external）樣式表**
  → 三種樣式表的使用時機、樣式**優先順序（precedence）**、瀏覽器預設樣式的影響。
- **2.2** 撰寫並分析 **CSS 規則集（rule set）**
  → rule set 的有效語法（selector + declaration block）；**選擇器**：class、id、element、pseudo-class、descendant（後代選擇器）。

### 領域 3：以 HTML 建構文件結構（3.1–3.4）
- **3.1** 撰寫並分析**結構化內容與組織資料**的標記
  → 表格（`table` / `tr` / `th` / `td`）、標題 `h1`–`h6`、`p` / `br` / `hr`、`div` / `span`、清單 `ul` / `ol` / `li`。
- **3.2** 撰寫並分析使用 **HTML5 語意元素**的標記
  → `header`、`nav`、`section`、`article`、`aside`、`footer`、`details`、`summary`、`figure`、`caption`（`figcaption`）。
- **3.3** 撰寫並分析實作**導覽（navigation）**的標記
  → `a href`、`target`、書籤（bookmark／頁內錨點）、相對 vs 絕對連結、資料夾階層路徑、圖片對應 `map` / `area`。
- **3.4** 撰寫並分析使用**表單元素**的標記
  → `form`（`action`、`method`、送出方式）、`input` 類型與限制、`select`、`textarea`、`button`、`option`、`label`。

### 領域 4：以 HTML 呈現多媒體（4.1–4.2）
- **4.1** 撰寫並分析**顯示影像**的標記
  → `img` 與 `picture` 元素及其屬性（`src`、`alt`、`width`/`height`、`srcset`）。
- **4.2** 撰寫並分析**播放影音**的標記
  → `video` / `audio` / `source`、`track`（字幕）、`iframe`（嵌入外部內容）。

### 領域 5：以 CSS 設定網頁樣式（5.1–5.4）
- **5.1** 撰寫並分析**定位內容**的樣式
  → 定位（`float`、`position: relative/absolute/static/fixed`）、`max-width`、`overflow`、`height`/`width`、`align`、`display`（inline vs block）、`visibility`、**盒模型（box model）**、`margin` 與 `padding`。
- **5.2** 撰寫並分析**格式化文字**的樣式
  → 字型（`font-family`、`font-style`、`font-size`、`font-weight`、`font-variant`）、`color` 與連結色、文字對齊與裝飾（`text-align`、`text-decoration`）、縮排、`line-height`、`word-wrap`、`letter-spacing`。
- **5.3** 撰寫並分析**格式化背景與邊框**的樣式
  → 邊框（`border-color`、`border-style`、`border-width`）、背景屬性與顏色（`background-color`、`background-image` 等）。
- **5.4** 撰寫並分析建立**簡易響應式版面**的樣式
  → 度量單位（`%`、`px`、`em`、`vw`、`vh`）、`viewport` 與 **media query**、frameworks 與 templates、**breakpoints**、grids。

### 領域 6：無障礙、可讀性與測試（6.1–6.3）
- **6.1** 撰寫符合**業界最佳實務**、格式良好的 HTML 與 CSS 標記
  → 重用 rules 與 rule sets、註解（commenting）、web-safe fonts、跨平台可用性、**結構（HTML）與樣式（CSS）分離**。
- **6.2** 套用**無障礙原則**並評估內容無障礙性
  → 文字替代（`alt` text）、色彩對比與使用、字體易讀性（legibility）、tab order、文字可縮放（text resizing）、text hierarchy、`translate`。
- **6.3** 評估 HTML 與 CSS 標記的**結構完整性**
  → 除錯：語法錯誤、標籤不匹配（未閉合／巢狀錯誤）、cascading（層疊）問題的辨識與修正。

## 五、應考者應具備的能力清單（Checklist）

備考自評，能自信打勾者代表該點掌握：

- [ ] 能寫出正確的 `DOCTYPE` 與 `html`/`head`/`body` 骨架，並判斷標記是否 well-formed
- [ ] 會設定 `meta`（charset、viewport、description）與 `link`、`style`、`script` 等 head 元素
- [ ] 能說出 inline / internal / external 三種樣式表的差異與**優先順序**
- [ ] 看到多條 CSS 規則能判斷實際套用結果（含選擇器優先權與 cascading）
- [ ] 熟悉 class / id / element / pseudo-class / descendant 選擇器語法
- [ ] 能用 `table`、清單、標題與 `div`/`span` 正確組織內容與資料
- [ ] 能正確使用 HTML5 語意標籤（header、nav、section、article、aside、footer）
- [ ] 會寫相對／絕對連結、頁內書籤，理解資料夾階層路徑
- [ ] 能建構含各類 `input`、`select`、`textarea`、`label` 的表單並設定 `action`/`method`
- [ ] 會用 `img`/`picture` 顯示影像、`video`/`audio`/`source`/`track` 播放影音、`iframe` 嵌入
- [ ] 理解盒模型與 `margin`/`padding`/`border`，能用 `float`/`position`/`display` 定位內容
- [ ] 會格式化文字（font、color、text-align/decoration、line-height、letter-spacing）
- [ ] 能用 `%`/`px`/`em`/`vw`/`vh`、`viewport` 與 media query 做簡易響應式版面
- [ ] 理解無障礙原則（alt 文字、色彩對比、tab order、text resizing、text hierarchy）
- [ ] 能找出標籤不匹配、語法錯誤與 cascading 問題並修正
